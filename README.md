# 테트리스 (교육용)

HTML, CSS, JavaScript만 사용하는 브라우저 테트리스 게임입니다.  
빌드 도구나 외부 라이브러리 없이 바로 실행할 수 있도록 만들어졌습니다.

## 프로젝트 소개

| 항목 | 내용 |
|------|------|
| 목적 | 프론트엔드 입문자를 위한 테트리스 학습 프로젝트 |
| 기술 스택 | HTML, CSS, JavaScript (Vanilla) |
| 보드 크기 | 10열 × 20행 |
| 블록 종류 | I, O, T, S, Z, J, L (7종) |

게임 로직(충돌 판정, 낙하, 고정)과 화면 렌더링(CSS Grid)이 분리되어 있어, 단계별로 기능을 확장하기 쉽습니다.

## 실행 방법

### 로컬에서 실행

1. 이 저장소를 클론하거나 폴더를 다운로드합니다.
2. `index.html`을 더블 클릭하거나 브라우저 창에 끌어다 놓습니다.
3. **시작** 버튼을 누른 뒤 키보드로 플레이합니다.

### VS Code Live Server (선택)

VS Code에서 **Live Server** 확장을 설치한 뒤, `index.html`에서 우클릭 → **Open with Live Server**로 열 수 있습니다.

## 조작법

| 키 | 동작 |
|----|------|
| `←` (ArrowLeft) | 왼쪽 이동 |
| `→` (ArrowRight) | 오른쪽 이동 |
| `↓` (ArrowDown) | 한 칸 빠르게 내리기 (soft drop) |
| `↑` (ArrowUp) | 블록 회전 (충돌 시 취소) |
| `Space` | 즉시 낙하 (hard drop) |

모든 조작은 `canMove()` 충돌 판정을 통과할 때만 적용됩니다.

## 게임 오버

새 블록을 스폰했을 때 보드와 겹치면 게임 오버입니다. **재시작** 버튼으로 다시 플레이할 수 있습니다.

## 품질 점검 방법

배포 전 아래 항목을 순서대로 확인합니다.

### 1. 리소스 로드

1. 브라우저에서 게임을 엽니다.
2. 개발자 도구(F12) → **Network** 탭에서 `style.css`, `script.js`가 **200**으로 로드되는지 확인합니다.
3. **Console** 탭에 빨간 에러가 없는지 확인합니다.

### 2. 기본 플레이

| # | 확인 | 기대 결과 |
|---|------|-----------|
| 1 | **시작** 클릭 | 블록 자동 낙하 시작 |
| 2 | `←` `→` | 좌우 이동, 벽에서 멈춤 |
| 3 | `↑` | 회전, 벽/블록 충돌 시 취소 |
| 4 | `↓` | 1칸 빠른 낙하 |
| 5 | `Space` | 즉시 착지 후 고정 |
| 6 | 가로 한 줄 완성 | 줄 삭제, 점수 증가 |
| 7 | 보드 상단까지 쌓기 | 게임 오버 표시 |
| 8 | **재시작** | 보드·점수 초기화, 플레이 재개 |

### 3. Cursor 커맨드 (선택)

프로젝트의 `.cursor/commands/`에 QA·리뷰용 커맨드가 있습니다.

- `code-review` — 코드 구조 리뷰
- `qa-playtest` — 기능별 테스트 시나리오
- `bug-hunt` — 잠재 버그 탐색
- `release-check` — 배포 전 체크

## GitHub Pages 배포 방법

이 프로젝트는 정적 파일만으로 동작하므로 GitHub Pages에 바로 배포할 수 있습니다.

### 사전 조건

- GitHub 저장소가 **이 게임 폴더를 루트**로 가져야 합니다.  
  (`index.html`이 저장소 최상단에 있어야 함)

### 배포 절차

1. GitHub 저장소 → **Settings** → **Pages**
2. **Source**: `Deploy from a branch`
3. **Branch**: `main` / `/ (root)` 선택 후 **Save**
4. 1~2분 후 `https://jisu-kim-81.github.io/tetris-cursor-20260617/` 에서 접속합니다.

### 배포 후 확인

- 게임 보드와 점수가 표시되는지
- **시작** 후 조작이 동작하는지
- Console에 404 (CSS/JS 경로 오류)가 없는지
